# AI 小镇设计规划

## 1. 项目目标

构建一个可持续运行的 AI 小镇模拟系统，让多个自治 Agent 在共享世界中进行生活、协作、竞争、交易与叙事演化。系统需要同时支持：

- 社会模拟：居民具备身份、目标、记忆、关系与行为策略。
- 世界演化：时间、地点、资源、建筑、事件会持续变化。
- 任务执行：Agent 可以接受任务、拆解计划、执行动作并反馈结果。
- 可观测性：开发者可以查看状态、日志、关系网络和关键事件。
- 可扩展性：后续可以接入更多 Agent 类型、规则引擎、LLM、前端界面。

## 2. 产品愿景

AI 小镇不是单轮对话系统，而是一个“持续存在的社会环境”：

- 每个居民都有相对稳定的人设和长期目标。
- 居民会因为记忆和关系而形成不同决策。
- 小镇每天都会发生可追踪的故事。
- 开发者可以注入任务、危机、资源变化来观察群体行为。

## 3. 核心设计原则

### 3.1 长短期状态分离

- 短期状态：当前位置、当前任务、情绪、背包、行动队列。
- 长期状态：身份设定、技能、价值观、社会关系、长期记忆。

### 3.2 决策与执行解耦

- 决策层负责“想做什么”。
- 执行层负责“能不能做、如何做、结果如何落地”。

### 3.3 世界先于对话

Agent 的输出不能只停留在文本，必须尽量转换为对世界状态的显式修改。

### 3.4 事件驱动

世界变化以事件为核心进行广播与记录，方便追踪、回放、统计与调试。

### 3.5 可替换的智能内核

Agent 的“思考模块”应支持多种实现：

- 规则策略
- 有限状态机
- 基于 LLM 的规划器
- 混合式决策器

## 4. 系统总体架构

建议采用 5 层架构：

- World Layer
- Agent Layer
- Simulation Layer
- Cognition Layer
- Interface Layer

## 5. 核心实体

建议先定义：

- `Place`
- `Agent`
- `Item`
- `Task`
- `Event`
- `WorldState`

## 6. MVP 范围

- 一个小型地图：5 到 8 个地点
- 5 到 10 个居民 Agent
- 基础需求：吃饭、休息、工作、社交
- 基础动作：移动、对话、交易、接任务
- 简单资源循环：食物、木材、金币
- 基础事件日志
- 命令行运行模拟

## 7. 技术建议

初版建议 Python 实现，并保持与当前 `Xenoverse` 项目风格一致。

建议后续模块：

- `world/`
- `agents/`
- `memory/`
- `simulation/`
- `events/`
- `tasks/`
- `interfaces/`

## 8. 下一步建议

1. 定义数据模型。
2. 实现仿真主循环骨架。
3. 提供一个最小小镇样例配置。
